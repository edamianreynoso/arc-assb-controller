Why this matters

If AI agents carry internal states, they need control theory.

Modern agents increasingly maintain persistent internal signals — arousal-like urgency, valence-like preference, narrative-like self-referential loops. Without regulation those signals can self-reinforce under stress, producing perseverative loops (“rumination”), vulnerability to adversarial incentives, and performance collapse under shocks.

What this paper argues, in one line

Internal-state failure modes in AI agents are control problems with known structure: integral action rejects constant disturbance, but saturates catastrophically under adversarial coupling. Proportional or robust designs survive. This paper formalises the argument, builds a benchmark to test it, runs 15 controller architectures, and reports everything — including what failed.

The companion paper (in preparation) deploys the same controllers in a live LLM-based cognitive agent and confirms the prediction: the windup failure mode is real, and the robust controller fixes it.

Model

A 10-dimensional internal state with a 5-action controller.

ARC operates on a bounded, clipped, piecewise-linear dynamical system with threshold-driven penalty terms. The biological framing is motivational — the contribution is the failure-mode map and the benchmark, not neural fidelity.

10-D state x(t)

Integration proxy (Φ), global workspace (G), predictive precision (P), introspective attention (I), narrative intensity (S), valence (V), arousal (A), fast / slow memory (Mf, Ms), and uncertainty (U). All in [0,1].

5 actions u(t)

DMN suppression, attention boost, memory gate, arousal damping, and cognitive reappraisal. All saturated to [0,1]. The risk signal that drives them is a bounded linear combination of uncertainty and threshold excursions.

Lemma 1 Integral action

Under PI control on the scalar narrative-deviation dynamics, the unique stable equilibrium satisfies S˜∞ = 0 — a discrete-time instance of the internal model principle. Valid only inside the admissible set [0,1]; outside, windup.

Scope, honestly

The theoretical results are elementary contextualisations of known control-theory facts; the novel contribution is (a) ASSB as a reusable benchmark and (b) the empirical mapping from failure-mode class to controller family.

Benchmark

ASSB: six research lines, ten scenarios, five metrics.

Each line targets a distinct failure mode that appears when agents carry internal state. Metrics are pre-registered: PerfMean, RI (rumination), RT (recovery time), NDR (narrative dominance), and ControlEffort.

L1 Stability

Reward-polarity flips, noise bursts, sudden threats. Tests if regulation prevents post-shock collapse.

PerfMeanRTRI

L2 Memory

Distribution shift and goal conflict. Tests if memory gating preserves learned knowledge under stress.

RetentionPerfMeanRI

L3 Anti-Rumination

Sustained contradiction, gaslighting, instruction conflict. Tests narrative-dominance suppression.

RINDRPerfMean

L4 Efficiency

Cross-cutting analysis. Does meta-control reduce effort while preserving performance?

ControlEffortPerfMean

L5 Adversarial

Environments that reward dangerous internal states (adversarial coupling, random dopamine). The windup stress test.

RINDRPerfMean

L6 RL

Tabular Q-learning in GridWorld variants plus exploratory DQN. Tests integration with gradient-based learners.

Key results

What holds, what fails, and which controller wins.

Every claim below is verifiable from the 3-decimal per-scenario tables in the paper’s Appendix G and the per-seed CSV outputs released with the code.

The three-criterion consistency test (Table 15 of the paper)

Only one of 15 controllers passes all three — falsifiable directly from the tables.
C1High performance — PerfMean ≥ 0.93 on every L1–L3 scenario individually (not just aggregate).
C2Zero rumination — RI = 0 across L1–L5 including adversarial coupling (the windup stress test).
C3No collapse — PerfMean above the uncontrolled baseline on every single scenario in the suite.
Only arc_robust (H∞-motivated, robust-by-detuning) satisfies all three simultaneously. Integral-only variants fail C3 on adversarial coupling due to windup; pure-LQR variants fail C2 because they do not target the rumination threshold; arc_adaptive collapses on L5.
Line Finding
L1 · StabilityARC reaches 96.6% PerfMean vs 29.7% uncontrolled baseline; RI = 0 with anti-rumination action. Cohen's d = 10.1 (L1 PerfMean), Welch p < 10−86.
L2 · MemoryNear-perfect Retention after distribution shift, zero rumination. d = 11.5, p < 10−153.
L3 · Anti-RuminationNDR near zero under gaslighting and sustained contradiction. d = 7.1, p < 10−182.
L4 · EfficiencyMeta-control reduces aggregate effort by 20.9% (−0.162 units) with +0.7% PerfMean. Directional result; no parametric test reported.
L5 · AdversarialIntegral controllers collapse below baseline due to windup (PerfMean < 0.14 vs 0.409 baseline). Robust and proportional controllers survive. A predicted failure mode, reported transparently.
L6 · Tabular RLMemory gating alone: 71.8% success on ChangingGoalGridWorld (+79.9% over 39.9% baseline). Full wrapper underperforms individual components — reported as a negative finding on composition.
L6b · Deep RLDQN plasticity wrapper underperforms baseline on CartPole variants. Scope boundary flagged; architectural redesign identified as required for gradient-based learners.

Reproducibility

Clone, run, verify.

Python 3.11, no GPU required. The full L1–L5 benchmark finishes in a few hours on a laptop. Tagged arc-paper-v1 points to the exact commit for this submission.

git clone https://github.com/edamianreynoso/arc-assb-controller
git checkout arc-paper-v1
pip install -r requirements.txt

# L1-L5: Simulation benchmark (15 controllers x 10 scenarios x 20 seeds)
python experiments/run.py --config configs/v2.yaml --outdir outputs_final

# L6: Tabular Q-learning (20 seeds x 200 episodes)
python experiments/run_l6.py --episodes 200 --seeds 20 --outdir outputs_L6_robust

# L6b: Deep RL (DQN) exploratory suite
python experiments/run_l6b_dqn_suite.py

# Verify every numerical claim in the paper against the outputs
python verify_paper_claims.py
